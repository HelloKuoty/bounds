# 主持人对照表 · Facilitator's Key（仅团队内部 · 绝不进游戏）

> 给想把《界 / Bounds》当"领域建模破冰"的 TechLead / 主持人用。
> **铁律:这张表永远留在游戏之外。** 玩家在游戏里只见寓言之名,绝不见下面右两列的词——
> 那声"哦,原来这就是……"的顿悟,要在通关后的复盘里由你来点,而不是写在屏幕上。
> （这也是为什么本文件放在 `team/` 而非 `data/` 或 `docs/`:禁词扫描只查游戏内文案,这里是台下讲稿。）

## 怎么用（一顿午饭的流程）
1. 让新人先**裸玩**教学主干（约 7 关），你一个术语都别解释，只让他亲手体会"名撞了得画界、一束太大要拆"。
2. 通关后开复盘，**一关一行**对照下表:先问他"你刚那步在解决什么?"，再用最右列那句话点破。
3. 收尾抛一个你们真实系统里的问题:"我们这儿，哪两个'账'其实是同名不同意?"——破冰完成。

## 对照表

| 游戏里的名（玩家所见） | 真实概念（复盘才说） | 复盘时你说的那一句 |
|---|---|---|
| 名 / 真意 | 统一语言 · 一词多义 | "同一个词，两个部门各表一意——这正是要先对齐的语言。" |
| 画界（把同名分到界外） | 划定边界 | "与其逼全公司共用一个'账'，不如划条界:界内一套说法，界外另一套。" |
| 成束 + 守门人 | 一致性整体 + 统一入口 | "一组必须一起改、要守同一条规矩的东西，立一个'守门人'统一进出。" |
| 拆束（束太大有代价） | 别让整体过大 | "守门人管太多就成瓶颈——大的要拆小，各守各的。" |
| 立译者石（连接两片界） | 翻译/隔离层 | "两个界要往来，中间架一层翻译，别让对面的说法脏了你这边。" |
| 誊本（把筹码誊往另一区） | 可复制的'值' | "这种东西不认身份，谁要谁拷一份，不必共用同一个。" |
| 共享（把活物借给另一区，有代价） | 共用的那一小块 | "真要共用的核心，省事，但从此两边一动都得商量——这是有代价的。" |
| 陈腐蔓延 / 土地塌陷 | 放任耦合的下场 | "边界不立、放任纠缠蔓延，最后就是谁也不敢动的一团泥。" |
| 心力（动作有成本） | 改造的认真预算 | "重整不是免费的——精力有限，挑要紧的边界先立。" |
| 此地之事 → 别处响应 | 以事件解耦 | "一处发生的事，广播出去让别处各自响应，而不是层层硬调。" |

## 现实翻车锚点（复盘时指着新人说:"你上次就栽在这"）
> 光有"名词→概念"会停在"哦原来这叫聚合"；配上一句真实事故钩子，才推到"我下周就改"。

- **名 / 真意**:上次"用户"这个词，风控说的和营销说的根本不是一回事，联调那天才发现对不上。
- **画界**:把所有东西塞进一个"订单"模型，结果售后改个字段、支付那边就炸——边界早该划。
- **成束 + 守门人**:库存谁都能直接改，对账永远对不平——没有守门人，人人都能伸手。
- **拆束**:那个"用户中心"什么都管，改一行要全公司回归——这就是一束揽太多。
- **立译者石**:把第三方的字段结构直接搬进核心，对方一改版，我们整条链路跟着重写。
- **誊本**:把"汇率"这种值做成强一致的共享服务，纯属自找瓶颈——拷一份快照就够了。
- **共享**:两个团队图省事共用一张表，半年后谁都不敢加索引——共享的债迟早要还。
- **陈腐蔓延 / 塌陷**:那个谁都不敢碰的老模块，就是当年没立边界、一点点烂成的。
- **心力**:想一口气重构整个系统，半年没上线还把自己拖垮——精力得用在刀刃的边界上。
- **此地之事 → 别处响应**:下单后要发短信、加积分、通知仓库，全塞进下单接口里，这接口最后没人敢动。

## 复盘引导脚本（一关一停 · 主持人照着带节奏）
> 用法:玩家通关后,你带着重玩(或回看截图)。每到一关,先让玩家用自己的话说"我刚做了什么",再抛下面这一句,把话题拽到他们自己的系统上。**问句优先于讲解**——让他们自己说出答案。

1. **渡口集市(画界)** — 玩家把同名两意的东西分到两片地。
   → 问:"我们系统里,哪个词在两个团队嘴里其实是两个意思?上次因此对错过吗?"
2. **账房(成束 + 守门人)** — 玩家把必须一起改的一簇收进一束、立了守门人。
   → 问:"我们哪一组数据是'必须一起改、却谁都能单独动'的?若只留一个入口,该是谁?"
3. **档案窖(拆束)** — 玩家把揽得太大的一束拆小。
   → 问:"我们有没有一个'什么都管'的模块,改它要全员回归?它该从哪一刀拆开?"
4. **两种话(译者石)** — 玩家在两片各说各话的地之间架了翻译。
   → 问:"我们和哪个上游/第三方,字段对不上、各论各的?中间那层翻译现在谁在背?"
5. **两市(誊本)** — 玩家把一个'不认身份的值'拷一份过去,而非强求共用。
   → 问:"我们有没有把一个本可各存一份的'值',硬做成了要强一致的共享服务?"
6. **共井(共享,有代价)** — 玩家让独一份的活物两边共用,代价是持续渗漏。
   → 问:"我们哪一块是真·共用、动一下两边都得知会?这笔'每次都要商量'的成本,值不值?"
7. **渗坏(遏制蔓延)** — 玩家趁腐烂没漫开,把好东西围到界外。
   → 问:"我们有没有一块正在烂、却没人敢碰的地方?现在能先把哪些好东西从它旁边隔开?"
8. **名障镇(认形色,莫认名)** — 玩家发现:同名未必同物,异名未必异物。
   → 问:"我们有没有两个名字一样、其实该拆开的东西?又有没有两个名字不同、其实是一回事、该合起来的?"
9. **蔓沼(收尾)** — 边界全塌、名全混的烂摊子,玩家重新切开。
   → 问:"如果给你一天什么都不上线,只重画我们系统的边界,你第一刀切哪儿?"

> 收尾别急着总结术语。让最后那个问题悬着,带回工位——破冰的目的不是"教完",是让他们下周看自己代码的眼神变一点。

## 一句话总纲
> 整局只在做一件事:**在混沌蔓延之前，把'谁和谁该在一起、谁和谁该隔开'划清楚。** 这，就是建模。
