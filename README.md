# 界 / Bounds

> 一个**寓言式**单人策略解谜 Roguelike-lite。你走入一片名实交错、正在渗坏的国土,
> 靠"分清名相、收拢整体、接通往来、围住腐烂"把它一处处理出秩序。
>
> 玩家从不听到一个软件术语 —— 但整个游戏的"物理定律",就是 DDD(领域驱动设计)的灵魂。
> **会玩 = 会建模;我们从不教,让思想藏在玩法里、让玩家自己悟。** 详见 [DESIGN.md](DESIGN.md)。

## ▶ 在线试玩(浏览器直接玩,无需安装)

### **https://hellokuoty.github.io/bounds/**

打开即玩。点棋子选中 → 按动作按钮。目标:化解一片地里所有**失稳**、填满「秩序」,别让「陈腐」先漫满。
顶部引导会随局势教你**用哪个动作、怎么诊断**(从不直接点破是哪几枚棋子);学会后逐步淡出,真卡住才递一句类型提示。

## 玩法一览

每片地是若干**棋子**(各有汉字之「名」、形状徽记之「含义」、颜色),散在一片或多片**区域**里。
六个结构动作各耗一点「心力」:

| 动作 | 解决的"谬" | 寓意 |
|---|---|---|
| **画界** | 同名异意 / 腐斑漫开 | 划清边界,让一个名只指一件事 / 把好的围到界外 |
| **成束** | 本是一体却散着 | 把同属一体的收归一束、立个守门人 |
| **拆束** | 一束揽得太多 | 把臃肿的一束分门别类 |
| **立译者石** | 界两边该往来的断了 | 在边界上架一座翻译的通路 |
| **誊本** | 缺一样(可誊的) | 复制一份无名之物补上 —— 但副本会"走味" |
| **共享** | 缺一样(独一份) | 两处合用一样独一物 —— 但从此一同渗坏 |

进阶机制:**传令链**(一道令顺信使一环环传到头,半路被界隔断就失效,需断处架译者石)、
**固/锚棋**(有些棋子生根钉死、画界挪不动,只能绕着它把别的分出去,甚至逼出"明知次优也得供着"的全局取舍)。

## 一局结构(Roguelike-lite)

教学骨干(逐个引入动作,不可跳过)→ 传令所(学传令链)→ **分叉层**(几条各有性格的真关,选项每局洗牌)→
**强制随机试炼**(先选「心法」:博观 / 持重 / 通变 —— 横向取舍非难度;局面随种子而变、多类失稳交织)→ 腹地 Boss。
清关按用时给 **★/★★/★★★**,并附一句随地名的微旁白(落情感 + 点回刚用的手之法)。

## 完成度 / 无障碍

- 失稳以**蔓延的墨晕病灶**呈现(非弹标点);严重度靠墨渍范围读 —— **离颜色、离动效**亦可读。
- 选中一枚棋子,**同名的一同亮起**(不识汉字也看见名组);选中 / 同名 / 普通靠**边宽阶梯**区分,不靠色相。可关动效。
- 美术 / 音效为**程序化合成**(色块几何 + 水墨着色器 + 合成音 + 清关磬),无外部资产,网页可跑。

## 状态

**功能完整、可一局 15–25 分钟通关。** 核心棋盘模型、六动作 + 传令链 + 固锚、节点地图、随机试炼生成器、
存档与 ★ 记录、程序化视听皆就绪;**测试 136 项常绿、全程烟雾测试通关**。零术语铁律由 `tests/test_no_jargon.gd` 守。
